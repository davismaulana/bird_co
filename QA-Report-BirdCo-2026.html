SEO Issues

HIGH No Hreflang Tags

Search engines cannot discover alternate language versions. This severely impacts multilingual SEO.

Required:

<link rel="alternate" hreflang="fr" href="https://birdandco.fr/fr/..." />
<link rel="alternate" hreflang="en" href="https://birdandco.fr/en/..." />

HIGH No Canonical URL

Both /en/privacy-policy and /en/politique-de-confidentialite are indexable as separate pages with identical content.

Required: <link rel="canonical" href="..." />

HIGH No Meta Description

The primary <meta name="description"> tag is completely missing.

Search engines may auto-generate descriptions or show irrelevant snippets.

MEDIUM Static Page Title

The <title> tag never changes per page or language. All browser tabs show "bird&co".

Recommended: Dynamic titles like "CFO Part-Time | Bird&" or "Services M&A | Bird&"

Recommended SEO Improvements

  • Add dynamic <meta name="description"> per page
  • Implement hreflang tags for FR/EN alternates
  • Add canonical URL tags
  • Update document.title dynamically per route
  • Update og:title and og:description per language
  • Create proper social sharing images (not favicon)
  • Implement structured data (JSON-LD) for business
  • Add sitemap.xml generation

BIRD&CO QA REPORT Section 9 - Critical & High Issues

9. Critical & High Priority Issues

CRITICAL #1: AppContent Component Defined Inside Render

Location: App.tsx:161

Description: The AppContent component is defined inside the App component's body. This creates a new component identity on every render, causing React to unmount and remount the entire page tree on every state change.

Impact: Form inputs lose data, animations restart, scroll position resets, performance degradation.

Fix: Move AppContent outside of App as a standalone component, or inline the rendering logic directly.

HIGH #2: No Hreflang Tags

Location: index.html, App.tsx

Impact: Google may index only one language or treat FR/EN as duplicate content.

Fix: Dynamically inject hreflang tags in App.tsx for both languages on every route.

HIGH #3-4: Missing Canonical and Meta Description

Location: index.html

Impact: Duplicate content issues, poor search snippets.

Fix: Add dynamic canonical and meta description injection per page.

HIGH #5: Non-Prefixed Paths Show 404

Location: App.tsx:67-73

Description: Only root / triggers redirect. Visiting /faq or /service/xyz without language prefix shows 404.

Impact: Broken bookmarks, shared links, external links.

Fix: Add catch-all redirect for missing language prefix.

HIGH #6: Tailwind CSS via CDN

Location: index.html:23

Description: Tailwind loaded via CDN script, not bundled.

Impact: Not production-ready, larger bundle, no tree-shaking.

Fix: Install Tailwind via npm and configure with Vite.

HIGH #7-8: Translation Errors

Files: en/services.json, fr/team.json

Issues:

  • "Prets participatifs" mistranslated as "Mezzanine Loans"
  • Typo "benefice" should be "beneficie"

Impact: Incorrect financial terminology, grammatical errors visible to users.

HIGH #9: Meta Tags Hardcoded in French

Location: index.html:8-21

Impact: Social sharing from English pages shows French text. All tabs show generic title.

Fix: Dynamic document.title and OG tag updates per language/page.
