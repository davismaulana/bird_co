BIRD&CO QA REPORT v2 Section 3 - Before vs After

3. Before vs After Comparison

Architecture

Before

  • AppContent defined INSIDE App component
  • Created new component identity on every render
  • Caused full tree remount on state change
  • Form inputs lost data, animations restarted

After

  • AppContent defined OUTSIDE App component
  • Stable component identity across renders
  • No unnecessary remounts
  • State preserved correctly

SEO & Metadata

Before

  • Static title "bird&co" for all pages
  • No meta description tag
  • No hreflang tags
  • No canonical URLs
  • OG tags hardcoded in French

After

  • Dynamic titles per page/language
  • Meta descriptions for all pages
  • Hreflang tags for FR/EN + x-default
  • Canonical URLs dynamically set
  • OG tags update per language

Language Switcher

Before

  • Full page reload on switch
  • Hash fragments (#contact) lost
  • Query parameters (?utm=x) lost
  • Fragile string replacement for slugs

After

  • Client-side navigation (no reload)
  • Hash fragments preserved
  • Query parameters preserved
  • Proper path parsing for slugs

Routing

Before

  • /faq shows 404 (no lang prefix)
  • /FR shows 404 (uppercase)
  • Only root path redirected

After

  • /faq redirects to /fr/faq or /en/faq
  • /FR normalizes to /fr
  • All non-prefixed paths redirect

Translations

Before

  • "Il benefice" (typo in French)
  • "Mezzanine Loans" (wrong term)
  • "All Rights Reserved" in FR file
  • Hardcoded "P&L", "Cash Flow", "KPIs"

After

  • "Il beneficie" (correct French)
  • "Participating Loans" (correct term)
  • "Tous droits reserves" in FR file
  • Translated mockup labels
BIRD&CO QA REPORT v2 Section 4 - Fixed Issues Detail

4. Fixed Issues - Detailed Verification

Critical Fix

FIXED #1: AppContent Architecture

File: App.tsx

Change: Moved AppContent component definition outside of App component. Added props interface for pathname, pathWithoutLang, currentLang, and t function.

Verification: Component no longer remounts on state changes. Form data preserved, animations stable.

SEO Fixes (#2, #3, #4, #9, #13, #14)

FIXED Dynamic SEO Implementation

File: App.tsx

Changes:

  • Added PAGE_META object with titles/descriptions for all pages in FR/EN
  • Created updateSEOMeta() helper to update document.title, meta description, OG tags
  • Created updateHreflangTags() helper to add hreflang and canonical tags
  • Both helpers called in useEffect on pathname change

Routing Fixes (#5, #15)

FIXED Path Normalization & Redirects

File: App.tsx

Changes:

  • Added useEffect to normalize uppercase paths to lowercase
  • Added redirect logic for non-prefixed paths (e.g., /faq → /fr/faq)
  • Cross-language slug validation added for service pages

Translation Fixes (#7, #8, #16, #17)

FIXED Content Corrections

Files: fr/team.json, fr/common.json, en/services.json, fr/pages.json, en/pages.json, constants.tsx

Changes:

  • "benefice" → "beneficie" (French verb conjugation)
  • "All Rights Reserved" → "Tous droits reserves" (French copyright)
  • "Mezzanine Loans" → "Participating Loans" (correct financial term)
  • Added pnl, cashFlow, kpis, openingCash, closingCash translations
  • Updated constants.tsx to use t() for mockup labels

Language Switcher Fixes (#10, #11, #12, #22)

FIXED Improved Language Switching

File: components/LanguageSwitcher.tsx

Changes:

  • Now uses useNavigate() instead of window.location.href
  • Preserves location.hash and location.search when switching
  • Calls i18n.changeLanguage() for client-side language change
  • Improved translatePath() with proper segment-by-segment slug translation

UX Fixes (#18, #19, #28)

FIXED Navigation & Animation Improvements

Files: App.tsx, components/Header.tsx, components/Notification.tsx

Changes:

  • Added scroll-to-top useEffect on pathname change (skips if hash present)
  • Fixed hash scrolling to work on consecutive clicks
  • Added delayed unmount to Notification for exit animation

Code Quality Fixes (#20, #21, #27)

FIXED Cleanup & Best Practices

Files: App.tsx, index.html, .gitignore

Changes:

  • Replaced window.__revealCleanup with useRef (no global pollution)
  • Removed import map from index.html (Vite handles React bundling)
  • Added .env, .env.local, .env.*.local to .gitignore

Recommendations for Remaining Issues

Issue #6: Tailwind via CDN

When to fix: Before production deployment or when optimizing bundle size.

Steps:

  • Install: npm install -D tailwindcss postcss autoprefixer
  • Run: npx tailwindcss init -p
  • Configure tailwind.config.js with content paths
  • Create src/index.css with @tailwind directives
  • Remove CDN script from index.html
  • Import CSS in index.tsx

Other Low Priority Items

These can be addressed during regular maintenance cycles or when refactoring related areas of the codebase.
